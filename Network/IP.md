### IP (Internet Protocol)

---

#### 0. 정의 
OSI 3계층(Network Layer)에서 동작하며, 패킷(데이터 조각)이 목적지까지 찾아갈 수 있도록 논리적인 주소(IP Address)를 부여하고 최적의 경로를 찾아주는(Routing) 규약.
#### 1. IP 주소 할당 (Address Allocation)
컴퓨터 네트워크에서 장치들이 서로를 인식하고 통신하기 위한 고유 식별 번호.

- **IPv4 vs IPv6**
  - **IPv4:** 현재 가장 많이 쓰이는 방식. 32비트 체계 (예: `192.168.0.1`). 주소 고갈 문제 발생.
  - **IPv6:** 주소 부족 해결을 위해 만든 128비트 체계. (예: `2001:0db8:...`).
- **공인 IP vs 사설 IP**
  - **Public IP (공인):** 전 세계에서 유일한 주소. 인터넷 통신 시 필수.
  - **Private IP (사설):** 공유기 내부(LAN)에서만 쓰는 주소. 외부에서 직접 접근 불가.
    - *대표 대역:* `192.168.x.x`, `10.x.x.x` (사내망 구축 시 주로 사용)

#### 2. 서브네팅 (Subnetting)
하나의 네트워크 대역을 효율적으로 관리하기 위해 여러 개의 작은 네트워크(Subnet)로 쪼개는 기술.

- **서브넷 마스크 (Subnet Mask):** IP 주소에서 어디까지가 '네트워크 ID'이고 어디서부터가 '호스트(기기) ID'인지 구분하는 기준.
- **CIDR 표기법:** `/24`, `/16` 등으로 표기.
  - *예시:* `192.168.0.10/24`
    - `192.168.0`까지가 우리 집 네트워크 주소.
    - `.10`이 내 컴퓨터 번호.
    - `/24`는 앞의 24비트(3자리)가 고정이라는 뜻 (255.255.255.0).

#### 3. 라우팅 (Routing)
출발지에서 목적지까지 데이터를 보낼 때 "어느 길로 가야 가장 빠른가?"를 결정하고 중계하는 과정.

- **라우터 (Router):** 서로 다른 네트워크(예: 우리 집 LAN ↔ KT 인터넷망)를 연결해 주는 장비.
- **라우팅 테이블 (Routing Table):** 목적지로 가는 지도(경로 정보)가 적힌 리스트.
- **게이트웨이 (Gateway):** 다른 네트워크로 나가기 위한 관문(출입구). 보통 공유기 주소(`192.168.0.1`)가 이에 해당.
