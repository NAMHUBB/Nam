### TCP (Transmission Control Protocol)

---
#### 정의
인터넷상에서 데이터를 메세지의 형태로 보내기 위해 IP와 함께 사용되며,
데이터의 신뢰성 있는 전송을 보장하기 위해 만들어진 연결 지향적 규약.

#### 특징

TCP는 '신뢰성'을 최우선으로 하는 메커니즘을 가짐.

- **연결 지향 (Connection-oriented**): 데이터를 주고받기 전, 송신측과 수신측 사이에 논리적인 연결 통로를 설정. (3-way Handshake)

- **신뢰성 보장 (Reliability)**: 패킷 손실 시 재전송을 요청하며, 데이터의 무결성을 보장.

- **순서 보장 (Ordering)**: 데이터가 각각 다른 경로로 전송되어 순서가 뒤바뀌더라도, 수신측에서 순서 번호(Sequence Number)를 통해 원래대로 재조립.

- **흐름 제어 (Flow Control)**: 송신측과 수신측의 데이터 처리 속도 차이를 해결하여 수신측의 버퍼 오버플로우를 방지.

- **혼잡 제어 (Congestion Control)**: 네트워크 내의 패킷 수가 과도하게 증가하여 발생하는 정체를 방지.

#### TCP의 동작 방식

연결 설정 및 해제

- **3-way Handshake (연결 시작)**: SYN -> SYN-ACK -> ACK 과정을 통해 상호 연결을 확정.

- **4-way Handshake (연결 종료)**: 데이터 전송이 끝나면 FIN과 ACK를 주고받으며 안전하게 연결을 해제.

#### 장점과 단점

- 장점
  
  - **데이터 무결성**: 패킷 손실, 중복, 순서 바뀜을 완벽히 해결

  - **안정적 통신**: 수신자의 처리 능력과 네트워크 혼잡도에 따라 속도를 조절

  - **에러 검출**: 체크섬(Checksum)을 통해 전송 중 데이터 변조 확인

- 단점
 
  - **속도 저하**: 연결 설정(Handshake) 및 확인 응답(ACK) 대기 시간 발생

  - **오버헤드 발생**: 패킷마다 번호를 매기고 관리하는 추가 제어 정보 필요

  - **지연 시간(Latency)**: 실시간성이 중요한 서비스에서는 불리함
